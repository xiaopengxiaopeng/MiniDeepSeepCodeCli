## 核心设计原则

### 1. 循环不变性
核心循环是 `while True:`（或等价形式）。从 s01 到 s08，这个循环**从未改变**。每一层机制只是在这个循环中插入一个薄薄的检查点。

### 2. 薄层叠加
每个机制是一个独立模块，在循环的特定时刻被调用：
- **执行前**：s06（压缩）、s04（提醒）
- **执行前**：s03（权限检查）
- **执行时**：s02（工具分发）、s07（错误重试）
- **执行后**：s05（收集子智能体结果）

### 3. 组合而非继承
没有一个庞大的基类。每个组件是独立的，被智能体通过组合持有：

```python
class FullAgent:
    def __init__(self):
        self.fs = VirtualFS()          # s02
        self.tool_map = build_tools()   # s02
        self.permissions = PermissionGate()  # s03
        self.todo = TodoManager()       # s04
        self.subagents = SubagentManager()  # s05
        self.compactor = ContextCompactor() # s06
        self.error_handler = ErrorRecovery() # s07
        self.llm = MockLLM()            # s01
```

## 每层机制的详细说明

### s01：智能体循环
```python
for query in queries:        # 主循环
    # s06: 压缩检查
    # s04: 提醒检查
    plan = llm.parse(query)   # LLM 解析
    # 路由：子智能体 / 计划 / 单工具
    # 观察 + 更新上下文
```
循环本身只有 **8 行**。其他一切都在循环体内被调用。

### s02：多工具调度
```python
tool_map = {
    "read_file":  lambda p: tool_read_file(fs, p),
    "write_file": lambda p: tool_write_file(fs, p),
    "edit_file":  lambda p: tool_edit_file(fs, p),
    "glob":       lambda p: tool_glob(fs, p),
    "bash":       lambda p: tool_bash(p),
}
```
添加新工具 = 1 个函数 + 1 个字典条目。循环不变。

### s03：权限管道
三个串行层级：
1. **拒绝列表** — 硬阻止匹配项（如 `rm -rf /`）
2. **破坏性检测** — 正则模式匹配（如 `\brm\b`）
3. **用户确认** — 仅在非自动批准时触发

### s04：Todo 写入
- `add_task()` — 分解复杂计划
- `mark_done_by_hint()` — 步骤执行后自动标记
- `nag_if_stale()` — 如果任务停留超过阈值则打印提醒

### s05：子智能体
- `Subagent.run()` — 带有干净上下文的迷你智能体循环
- `SubagentManager.collect()` — 批量收集结果
- 并行任务自动派生子智能体

### s06：上下文压缩
| 层级 | 触发条件 | 操作 |
|------|---------|------|
| snip | 历史 > 60 条 | 丢弃前半部分 |
| micro | 历史 > 30 条 | 汇总前半部分 |
| budget | 预算 > 硬限制 | 仅保留最近 5 条 |

### s07：错误恢复
```
尝试 1 ──失败──→ 等待 0.5 秒
尝试 2 ──失败──→ 等待 1.0 秒
尝试 3 ──失败──→ 等待 2.0 秒
─────────────────→ 触发 budget 压缩
```

## 学习路线

1. **阅读代码**：从 `python/code.py` 开始，它注释最详尽
2. **跟踪一次查询**：看一条简单查询（如 `read demo/file1.txt`）如何流经所有 7 层
3. **跟踪一个复杂查询**：看 `build a calculator app` 如何触发 todo、子智能体、多步骤
4. **修改工具集**：在 `build_tool_map` 中添加一个工具（如 `grep`）
5. **修改权限**：向 `DENY_LIST` 添加你自己的模式
6. **比较语言**：看看相同的架构如何用 Python、C++ 和 Java 表达

## 关键启示

> **"Many mechanisms, one loop."**
>
> 一个优秀智能体的力量不在于复杂的控制流，而在于能干净组合的薄层。每个机制解决一个特定问题（安全性、内存、可靠性），而不改变核心的 "观察 → 规划 → 行动" 循环。
>
> 如果你能理解这个单一循环的架构，你就能构建任何 AI 驱动的开发工具。
